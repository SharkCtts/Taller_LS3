Ejes Juan Carlos OS

Propósito

Herramienta para gestionar el inventario de repuestos automovilísticos en el taller mecánico Ejes Juan Carlos.

Resumen

El taller mecánico "Ejes Juan Carlos" enfrenta actualmente desafíos en el control, registro y actualización de su inventario de repuestos. Las tareas se realizan de forma manual o con herramientas genéricas como hojas de cálculo, lo cual genera errores, duplicidad de información y pérdida de tiempo en las búsquedas o actualizaciones de stock.

En el contexto específico del negocio, la falta de una herramienta especializada en gestión de repuestos impide un seguimiento eficiente de las entradas, salidas y actualizaciones de productos, lo cual afecta directamente la toma de decisiones, el control de costos y el nivel de servicio al cliente.

Se propone el desarrollo de una herramienta digital llamada Ejes Juan Carlos OS, un sistema de gestión diseñado especialmente para el taller “Ejes Juan Carlos”, que permita agregar, eliminar, actualizar y visualizar repuestos automovilísticos. La solución incluirá una interfaz web amigable, un backend organizado y una base de datos estructurada, todo desplegado en un repositorio privado con instrucciones detalladas para su sitio de uso.

Los beneficios de implementar esta solución incluyen el ahorro significativo de tiempo administrativo, una mejor precisión en el control de stock, reducción de pérdidas económicas por errores de registro, mejora en la atención al cliente al tener visibilidad inmediata del inventario, y la posibilidad de escalar el negocio con bases tecnológicas sólidas.

Objetivo General

Desarrollar un sistema de gestión que permita administrar de buena manera el inventario de repuestos automovilísticos en el taller “Ejes Juan Carlos”.

Objetivos Específicos

  • Analizar:
    • Req. Funcionales:
      • El Sistema debe permitir registrar nuevos repuestos con datos como nombre, categoría, cantidad y precio
      • El Sistema debe permitir modificar o eliminar repuestos existentes
      • El Sistema debe mostrar una lista actualizada del inventario a tiempo real, con opciones de búsqueda mediante filtros
      • El Sistema debe permitir visualizar gráficos sobre datos del inventario, tal como puede ser las ganancias mensuales
    • Req. No Funcionales:
      • El Sistema debe responder a las consultas de inventario de forma rápida
      • La interfaz debe ser intuitiva y fácil de usar
      • El sistema debe estar desarrollado con una arquitectura modular que permita agregar nuevos módulos

Desarrollar

Desarrollo Ritmit (paso a paso para desplegar en GitHub):

  1. Clonar el repositorio (opcional, para desarrollo o mantenimiento):
    git clone https://github.com/SharkcCtts/Taller_LS3.git
  2. Revisar la estructura del proyecto:
    • Verificar carpetas de frontend y backend (si aplica)
    • Revisar archivos de configuración para despliegue, en éste caso: vercel.json
  3. Configurar entorno local (solo para desarrollo):
    • Crear entorno virtual: python -m venv env
    • Activar entorno: source env/bin/activate (Linux/Mac) o env\Scripts\activate (Windows)
    • Instalar dependencias: pip install -r requirements.txt
  4. Ejecutar proyecto localmente (opcional):
    • Para backend: python manage.py runserver (Django) o comando equivalente Flask
    • Para frontend: npm install y npm run dev o el script definido
  5. Preparar actualización y despliegue en Vercel:
    • Hacer commits con cambios en el código
    • Push a GitHub (Vercel suele integrarse automáticamente con GitHub)
    • Vercel detecta el cambio y despliega automáticamente
  6. Verificar despliegue:
    • Abrir el link https://taller-ls-3.vercel.app/
    • Comprobar que las funciones (agregar, eliminar, actualizar productos) funcionan correctamente
  7. Uso para usuarios finales:
    • Solo abrir el navegador y entrar a https://taller-ls-3.vercel.app/
    • Iniciar sesión y usar la aplicación
  8. Monitoreo y mantenimiento:
    • Revisar logs de errores en la consola de Vercel
    • Actualizar código y redeplegar según sea necesario
